### Happy Path Tests

**1. `test_51.txt`: Simple Callable Object Call**
This test defines a `Multiplier` struct with a `call` function to multiply an input by a stored factor. It demonstrates the shorthand `m(10)` syntax for `m.call(10)`.

```
struct Multiplier
{
    i32 factor

    fn call = (i32 x) -> i32
    {
        return factor * x
    }
}

Multiplier m{5}
// Equivalent to m.call(10)
i32 result{m(10)}
return result
// Expected Result: 50
```

**2. `test_52.txt`: Callable and Member Function Equivalence**
This test demonstrates that both the explicit member function call (`v.call()`) and the callable object syntax (`v()`) work identically for the same `call` function.

```
struct Value
{
    i32 val

    fn check = () -> i32
    {
        return val + 1
    }

    fn call = () -> i32
    {
        return val + 1
    }
}

Value v{9}
i32 a{v.call()} // Explicit call
i32 b{v()}      // Callable object syntax
i32 result{a + b}
return result
// Expected Result: (9 + 1) + (9 + 1) = 20
```

**3. `test_53.txt`: Callable Object Modifying State (Mutable)**
This test uses a mutable object (`Counter mut c`) with a `call` function that modifies its internal state (`value`). Subsequent calls reflect the new state.

```
struct Counter
{
    i32 mut value

    fn call = (i32 x) -> i32
    {
        value = value + x
        return value
    }
}

Counter mut c{10}
i32 a{c(5)} // value becomes 15, returns 15
i32 b{c(2)} // value becomes 17, returns 17
i32 result{a + b}
return result
// Expected Result: 15 + 17 = 32
```
